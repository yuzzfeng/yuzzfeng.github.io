Topologische Qualitätsprüfung von OSM-Daten mit LLM-Unterstützung

Analyse und Reparatur von OSM-Topologiefehlern (z. B. unverbundene Wege, Überlappungen, fehlerhafte Relationen) mit Hilfe von Tools wie OSMose. Ziel ist eine Pipeline, die Issues automatisiert abruft, priorisiert und mit LLM-Hilfestellung (Vorschläge, Erläuterungen, Skripte für Korrekturen) behebt, inklusive Validierungsschritten und Dokumentation der Änderungen.

English Summary

This thesis focuses on detecting and repairing OpenStreetMap topology errors (e.g., disconnected ways, overlaps, and invalid relations). The project aims to develop a workflow that retrieves and prioritizes issues and uses LLM support for explanations, fix suggestions, and reproducible correction scripts.

Tools & Daten

  • OSMose — automatisierte OSM-Fehleranalyse
  • OpenStreetMap Overpass API
  • LLM API (z. B. OpenAI GPT, lokales Modell)

Anforderungen

  • Kenntnisse in OpenStreetMap und GIS
  • Programmierkenntnisse (Python)
  • Interesse an Datenqualität und LLMs
